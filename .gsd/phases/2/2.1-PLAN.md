# Phase 2.1 Plan: Chrome Extension Boilerplate & Manifest

> **Objective**: Migrate the project from a simple bookmarklet build to a Chrome Extension structure with a `manifest.json`, background service worker, and content scripts.

## Steps
1. **Create `manifest.json`**: Define extension name, version, permissions (`activeTab`, `storage`, `scripting`, `https://generativelanguage.googleapis.com/*`), host permissions, and background/action configurations.
2. **Setup Extension Architecture**:
   - `background.js` (Service Worker): Handles the API requests to Gemini to bypass content security policies on the host page.
   - `content.js`: Handles text extraction from the input fields and injecting the prompt review UI into the web page.
   - `popup.html` and `popup.js`: Move the settings (API Key, Improvement mode) from the injected settings modal into the extension's native popup, streamlining the UX.
3. **Update Build System**: Modify `scripts/build.js` to bundle these multiple entry points instead of just one `bookmarklet.js` file, copying `manifest.json` and static files to `dist/`.
4. **Update STATE.md**: Reflect the transition into Phase 2.

## Expected Outcome
The `dist` folder will contain a valid unpackaged Chrome Extension with the appropriate architecture files ready for logic migration.
