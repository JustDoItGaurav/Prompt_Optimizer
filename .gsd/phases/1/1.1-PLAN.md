# Plan 1.1: Adapters & Storage

## Objective
Establish the core foundation for reading/writing prompts across various AI playgrounds using the adapter pattern, and securely managing the `localStorage` settings.

## Context
- .gsd/SPEC.md
- .gsd/ROADMAP.md

## Tasks

<task type="auto">
  <name>Create Storage Helper</name>
  <files>src/storage.js</files>
  <action>
    Implement a minimalist utility to manage `localStorage`. It must support retrieving, saving, updating, and clearing a JSON payload. The payload should track the `geminiApiKey`, `improvementMode` (Quick/Standard/Expert), and explicit overrides for the target. Do NOT hardcode the key. All logic must defensively check `window.localStorage` existence.
  </action>
  <verify>Ensure `src/storage.js` exports reliable functions without syntax errors.</verify>
  <done>Storage wrapper is successfully created and handles read/write of API Key and Settings.</done>
</task>

<task type="auto">
  <name>Implement Extraction Adapters</name>
  <files>src/adapters/*</files>
  <action>
    Create the adapter structure: `src/adapters/index.js`, `generic.js`, `chatgpt.js`, `claude.js`, `gemini.js`. 
    Each specific adapter should implement `getPrompt()`, `setPrompt(text)`, and `getSiteName()`.
    - `generic.js` should implement the flexible fallback: search `textarea` then `[contenteditable="true"]` then `role="textbox"`, grab the largest one.
    - Important: writing text to these requires safely synthesizing input/change events for React UI compatibility.
  </action>
  <verify>Check that adapters are correctly mapped by hostname in `index.js` and that `setPrompt` dispatches real DOM events like `InputEvent`.</verify>
  <done>Adapter factory (`getAdapter()`) correctly resolves based on location, and all basic interfaces are formed.</done>
</task>

## Success Criteria
- [ ] Local storage helper works defensively.
- [ ] Generic adapter implements the fallback DOM extraction.
- [ ] Adapter index exports a resolver mapping hostname to correct adapter logic.
