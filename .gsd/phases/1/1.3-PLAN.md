# Plan 1.3: Main UI & Compilation

## Objective
Finalize the main floating bookmarklet UI that initiates the reframer flow, connects Settings + API Fetcher + Adapters together, and packages the result into a clean build.

## Context
- .gsd/ROADMAP.md Phase 1 MVP
- src/adapters/
- src/settings.js
- src/api.js

## Tasks

<task type="auto">
  <name>User Interface Orchestration</name>
  <files>src/main-ui.js</files>
  <action>
    Construct the main `ReframerModal` handling the core interaction loop.
    1. Render a "Improve Prompt" window displaying a loading spinner during API fetches.
    2. Once `api.js` returns, hide spinner and display the improved text in a focused block.
    3. Include 3 Buttons: 'Copy' (navigator.clipboard.writeText), 'Replace', and 'Settings'.
    4. 'Replace' invokes the current adapter's `setPrompt(improvedText)` correctly.
    5. 'Settings' invokes `showSettings()` from `settings.js`.
    6. Ensure the modal can cleanly unmount (close button).
  </action>
  <verify>Ensure `ReframerModal` renders vanilla HTML nodes dynamically without global window variable leakage.</verify>
  <done>The UI elegantly handles wait states, triggers copying/replacing safely, and hooks to settings.</done>
</task>

<task type="auto">
  <name>Package Compilation Script</name>
  <files>src/index.js, scripts/build.js</files>
  <action>
    - `src/index.js` serves as the entrypoint. It detects current site adapter (`adapters/index.js`), extracts original text via `getPrompt()`, and if empty, alerts user. If text exists, checks `localStorage` for API key. If missing, triggers `settings.js`. Otherwise, mounts `main-ui.js` and calls API.
    - `scripts/build.js` should bundle all `src/*.js` files together via a simple node concat process (or lightweight bundler like esbuild/webpack) into a single `dist/bookmarklet.js` wrapped in `javascript:(function(){...})()`.
  </action>
  <verify>Run `node scripts/build.js` and check if `dist/bookmarklet.js` is generated successfully stringified.</verify>
  <done>Bookmarklet script compiles successfully replacing manual testing into a URI encoded format ready for bookmark bars.</done>
</task>

## Success Criteria
- [ ] UI orchestration properly ties all logic units together with user-friendly layout and buttons.
- [ ] Bookmarklet is generated and cleanly wraps the application in an IIFE.
