# Plan 1.2: Settings & API Fetcher

## Objective
Develop a lightweight Settings modal UI to manage the `localStorage` credentials, and pair it with the Gemini REST API caller taking input from the target site adapters.

## Context
- .gsd/SPEC.md
- src/storage.js
- src/adapters/index.js

## Tasks

<task type="auto">
  <name>Create Setting Modal</name>
  <files>src/settings.js</files>
  <action>
    Design a vanilla HTML/CSS modal. Elements: an input (`type="password"`) for the Gemini API key, select boxes for "Improvement Mode", plus Save / Close buttons.
    - Scope CSS strictly (e.g., `#prompt-reframer-settings-modal *`) to avoid bleeding onto host sites.
    - On initialization (`showSettings()`), read from `src/storage.js` and populate inputs.
    - On save, save to `localStorage` via the storage helper.
  </action>
  <verify>Ensure `showSettings` creates a detached DOM element, appends it dynamically, and handles Save events without relying on frameworks.</verify>
  <done>Settings UI is cleanly injectable and successfully ties to `src/storage.js`.</done>
</task>

<task type="auto">
  <name>Gemini API Integration</name>
  <files>src/api.js</files>
  <action>
    Export an async function `improvePrompt(originalText, settings)`.
    - Fetch the latest key via `storage.js`.
    - Use JavaScript's native `fetch()` to call `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent?key=YOUR_API_KEY` (use gemini 2.5 flash or similar depending on the exact URI pattern - we will assume standard text completion POST format).
    - Implement a System Prompt enforcing professional prompt engineering: "You are an expert prompt engineer. Restructure the user input into a refined request ensuring role, context, explicit steps, and output format. Preserve original intent exactly."
    - Return the improved string. Handle generic errors using try/catch.
  </action>
  <verify>Confirm `api.js` returns parsed JSON `.candidates[0].content.parts[0].text` safely.</verify>
  <done>The `improvePrompt` function returns restructured texts given an API key and original prompt.</done>
</task>

## Success Criteria
- [ ] Settings modal allows inputting and saving an active API Key locally without cloud dependency.
- [ ] API Fetcher constructs system requests enforcing structural engineering, returning improved strings.
